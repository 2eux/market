```bash

  "https://market.servo.host/api/trade/sell?ticker=XLU"
```

## 8. 🔧 Troubleshooting Notes

### API Key Format
The API keys are plaintext strings defined in `auth.py`:
```python
VALID_API_KEYS = {
    "market-trade-bot": {"role": "full_access", "name": "Trading Bot"},
    "market-web-dashboard": {"role": "readonly", "name": "Web Dashboard"},
}
```
**Important:** The `***` literal should NEVER be sent. Always use the exact key string.

### Common Issues
1. **"Unauthenticated"** — API key has `***` literal instead of actual key value
2. **"Insufficient funds"** — SPY is $746/share, $200 capital can only buy sub-$50 ETFs
3. **"Method Not Allowed"** — Using GET on POST endpoints (/trade/buy requires POST)
4. **App shows "exited" in Coolify** — Cosmetic, app runs fine. Coolify auto-deploy pipeline broken
5. **Quote returns error** — yfinance MultiIndex parsing was fixed, rebuild if persists

### Restart Commands
```bash
# Stop and restart market app

docker run -d --name a13aqk67azm9nboo5voasje9 --restart unless-stopped --network coolify \
```

## 9. 🛠 Key Decisions Made

1. **Coolify 4.1.2 over 4.1.0** — Needed for full docker-registry support. Auto-upgrade disabled.
2. **Manual Docker over Coolify deploy** — Build pipeline broken in 4.1.2 after DB migration. Manual `docker run` works.
3. **ghcr.io registry** — Local image accessible to Coolify if deploy pipeline is fixed.
4. **API key auth over JWT** — Simpler for now. Upgrade to JWT at $25k+ tier.
5. **Paper trade mode** — No live IBKR connection yet. All trades simulated.
6. **DB quote caching** — Avoids YFinance latency on every quote request.
7. **Sub-$50 ETFs** — Only viable instruments for $200 validation capital per MC-2.
